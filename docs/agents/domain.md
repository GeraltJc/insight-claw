# Domain Docs

本文档说明工程技能在探索本项目代码库前，应如何读取领域上下文和架构决策。

## 探索前优先读取

- 根目录 `CONTEXT.md`：项目领域语言、核心边界、业务规则和术语。
- 根目录 `CONTEXT-MAP.md`：如果存在，说明项目已切换为 multi-context 布局；按其中映射读取相关上下文。
- `docs/adr/`：读取与当前工作区域相关的架构决策记录。

如果这些文件或目录不存在，继续工作即可。不要因为缺失这些文件而阻塞任务，也不要在没有明确需求时主动创建。`/grill-with-docs` 可在术语或决策真正被确认时按需补充。

## 文件结构

本项目当前按 single-context 仓库处理：

```text
/
├── CONTEXT.md
├── docs/adr/
└── src/
```

## 使用项目术语

当输出中需要命名领域概念时，例如 issue 标题、重构建议、调试假设、测试名称，应优先使用 `CONTEXT.md` 中定义的术语。

如果需要的概念尚未出现在 `CONTEXT.md`，这可能表示两种情况：

- 正在使用项目尚未确认的语言，应重新检查。
- 项目确实缺少一条领域定义，可记录为后续 `/grill-with-docs` 的补充点。

## 标出 ADR 冲突

如果输出内容与已有 ADR 冲突，应明确指出冲突，而不是静默覆盖既有决策。
